# 컨셉추얼KV — 광고 키비주얼 생성

> 하나은행·리얼베리어 런에서 반려된 것들의 반대를 한다: 레퍼런스 그대로 베끼기 ❌, 카피가 PPT 텍스트처럼 밋밋 ❌, 시안 2개만 ❌, 스톡틱 ❌.

## 정체성 한 줄
**KV의 주인공은 이미지가 아니라 "디자인된 카피 타이포그래피"다.** 공식: 말장난 카피 × 무드 맞춤 타이포(대형, 화면 1/3~1/2) × 인물·오브제 상호작용 × 단색·시네마틱 배경 × 로고 고정.

## 프로세스

### 1. 브리프 확인
- 브랜드/제품, 타겟, 핵심 카피(있으면) 확인. 카피가 없으면 → **전용 카피 스킬이 있으면 그걸로 먼저 짓고**, 없으면 Claude가 후킹 원칙(라임·말장난·USP 뒷받침, 낚시 금지)에 따라 즉석에서 3~5개 지어 사용자에게 고르게 한다. 이 스킬 자체는 카피 완성도보다 비주얼 완성에 집중한다.
- 규제 업종(금융·의료 등)이면 확정 어구만, 과장 금지 표기.

### 2. 컨셉 발산 (최소 10개)
카피 하나를 그대로 그리지 않는다. 아래 레버로 **비주얼 컨셉 후보 10개**를 발산한다:
- **오브제 치환·강제원근** — 제품/USP를 다른 사물로 치환하거나 스케일을 비튼다 (미니어처 월드, 손+하늘 시리즈)
- **타이포 개입** — 대형 타이포가 이미지를 찢고/가르고 들어옴
- **위트 코드** — 유머 한 스푼 (구글리아이, 말장난 비주얼)
- **레퍼런스는 감도 학습용일 뿐** — 사용자가 레퍼런스를 줬어도 레이아웃·기법(찢기, 손글씨 등)을 그대로 복붙하지 않는다. 오리지널 컨셉으로 재구성.
- 10개 중 결이 다른 3~4개를 사람에게 보여주고 방향 게이트. ("알아서 쭉" 지시가 있었으면 게이트 생략하고 가장 강한 1~2개로 바로 진행.)

### 3. 무드 → 타이포 매핑 (반드시 프롬프트에 명시)
"bold headline/clean sans-serif"만 쓰면 밋밋한 고딕=PPT 텍스트로 나온다. 무드별 서체 처리를 프롬프트에 **아트디렉팅 문구로 직접 지정**한다:

| 무드 | 서체 처리 | 프롬프트 키워드 예 |
|---|---|---|
| 다크·프리미엄 | 크롬/리퀴드 메탈 + 글로우 | "chrome liquid metal typography, specular highlight, dark studio glow" |
| 위트·캐주얼 | 마커 손글씨 | "thick marker handwriting, slightly imperfect strokes, playful" |
| 감성·휴먼 | 붓 캘리그래피 | "brush calligraphy ink stroke, soft bleed edges" |
| 조형·임팩트 | 3D 압출 + 질감 | "3D extruded typography, beveled edges, dramatic rim light, material texture" |
| 미니멀·클린 | 라인과 하나된 얇은 타입 | "thin minimal line type integrated into composition, negative space" |
| 신뢰·정직(금융 등) | 3D 압출 단색(브랜드컬러) | "solid brand-color 3D extruded type, clean matte finish, no gradient" |

이펙트(번개·엠보싱·왜곡·그림자)까지 구체적으로 지정한다.

### 4. 실사 미장센 레버 (스톡틱 회피)
- 렌즈(85mm f/1.4), 조명(soft key + rim), 필름 그레인, 컬러 그레이딩(teal-blue editorial), 구도(negative space for copy)
- 프롬프트에 반드시: `"no stock-photo feel, high-end beauty campaign, Vogue/Kinfolk editorial grade"`
- 비율 왜곡 금지 — 원본 비율 유지, 셀에 안 맞으면 크롭이지 늘리기 아님.

### 5. 생성 (이미지 생성 커넥터 필요)
> ⚠️ **전제**: Claude 자체는 이미지를 만들지 못한다. 이 단계는 **이미지 생성 MCP/커넥터가 연결된 환경**(nano-banana·Higgsfield·Midjourney 브리지 등)을 전제로 한다.
> - 커넥터가 있으면 → `generate_image` 계열로 **최소 6개 시안** 생성.
> - 커넥터가 없으면 → **완성형 프롬프트 패키지(6벌)까지만 출력**하고, 사용자가 본인 이미지 툴에 붙여넣게 안내한다. (스킬은 여기까지가 자기 역할)
- 2개는 부족 — 반려 사유였음. 무조건 6개 이상.
- **이미지마다 한 줄 정리 필수**: ①어떤 컨셉인지 ②어떤 느낌으로 만들었는지 ③뭘 참고했는지 (레퍼런스가 있었다면).
- 로고 고정 위치(하단 또는 지정 위치) 반드시 합성. KV 하나에 컨셉 하나만 — 컨셉 없는 예쁜 무드샷은 실패.

### 6. 리뷰 게이트
- 사람에게 6개 중 픽 요청 (narrow 게이트 — 컨셉 방향까지 재질문하지 않음, 이미 골랐으면 바로 다음 단계).
- 원본 벤치마크가 있는 런(비딩 제안서 등)이면 **벤치마크의 구조·밀도를 그대로 계승**했는지 자체 점검 — 여백 많은 문서형으로 임의 재구성했으면 반려감.

## 이 스킬이 하지 않는 것
- 카피 완성도 자체(라임·말맛 다듬기)를 깊게 파는 일 → 전용 카피 스킬이 있으면 그쪽에 위임
- 스토리보드 컷 구성·영상 시퀀스 → 별도 진행(단, 같은 무드→타이포 매핑·실사 레버 원칙은 그대로 적용)

## 체크리스트 (출력 전)
- [ ] 카피는 확정된 것에서 왔는가, 비주얼 작업 중 즉흥으로 바꾸지 않았는가
- [ ] 컨셉 10개 발산했는가, 레퍼런스 그대로 베끼지 않았는가
- [ ] 타이포 무드 처리를 프롬프트에 구체적으로 넣었는가 (고딕 단독 금지)
- [ ] 최소 6개 시안 + 각 이미지 한 줄 정리
- [ ] 로고 고정 위치 반영했는가
- [ ] 비율 왜곡 없이 원본 비율 유지했는가
