# 컨셉추얼KV — 광고 키비주얼 생성

> 하나은행·리얼베리어 런에서 반려된 것들의 반대를 한다: 레퍼런스 그대로 베끼기 ❌, 카피가 PPT 텍스트처럼 밋밋 ❌, 시안 2개만 ❌, 스톡틱 ❌.

## 정체성 한 줄
**KV의 주인공은 이미지가 아니라 "디자인된 카피 타이포그래피"다.** 공식: 말장난 카피 × 무드 맞춤 타이포(대형, 화면 1/3~1/2) × 인물·오브제 상호작용 × 단색·시네마틱 배경 × 로고 고정.

## 프로세스

### 0. 오프닝 게이트 (발동 즉시 — 생성 전 필수, 건너뛰면 위반)
스킬이 발동하면 **가장 먼저** 아래 4가지를 사용자에게 묻는다(AskUserQuestion 1회로 묶어서). 이걸 안 묻고 바로 컨셉·이미지로 가면 "비전문가가 대충 만든" 결과가 나온다 — 실패 사유였음.
1. **사이즈/비율** — 세로 포스터 2:3 / 정사각 1:1 / 가로 와이드 16:9 / 스토리 9:16 / 세트. (규격이 레이아웃·타이포 배치를 결정)
2. **컨셉 방향** — 어떤 빅아이디어로 갈지 (모르면 발산 후 고르게).
3. **레퍼런스 유무** — 참고 이미지·무드보드가 있는지 (있으면 감도만 학습, 복붙 ❌).
4. **실사 무드 + 카피 디자인 수준** — 인물 무드(영화 스틸컷 / 패션 화보 / 다큐 리얼리즘)와 타이포를 어디까지 포스터급으로 디자인할지.
> 답을 받고 나서 1단계로 간다. ("알아서 쭉" 지시가 명시적으로 있었을 때만 합리적 기본값으로 이 게이트를 스킵할 수 있다.)

### 1. 브리프 확인
- 0단계 답 + 브랜드/제품/타겟/핵심 카피 확인. 카피가 없으면 → **전용 카피 스킬이 있으면 그걸로 먼저 짓고**, 없으면 Claude가 후킹 원칙(라임·말장난·USP 뒷받침, 낚시 금지)에 따라 즉석에서 3~5개 지어 사용자에게 고르게 한다. 이 스킬 자체는 카피 완성도보다 비주얼 완성에 집중한다.
- 규제 업종(금융·의료 등)이면 확정 어구만, 과장 금지 표기.

### 2. 컨셉 발산 (최소 10개)
> 발산 전에 **[references/kv-references.md](references/kv-references.md)를 먼저 읽는다.** 실제 광고 KV 보드(핀 500+) + Behance/인스타 관찰에서 뽑은 10개 유형(A 오브제 위트 · B 타이포 주인공 · C 강렬 원색 원샷 · D 에디토리얼 인물 · E 미니멀 프리미엄 · F 재질감 재해석 · G 캠페인 히어로/K-pop·테크뷰티 · H 매크로 오브제 스타일링 · I UI 카드 그리드 · J 럭셔리 컨셉추얼 CGI)과 앵커 사례가 감도 기준이다.

카피 하나를 그대로 그리지 않는다. 아래 레버로 **비주얼 컨셉 후보 10개**를 발산한다:
- **오브제 치환·강제원근** — 제품/USP를 다른 사물로 치환하거나 스케일을 비튼다 (미니어처 월드, 손+하늘 시리즈)
- **타이포 개입** — 대형 타이포가 이미지를 찢고/가르고 들어옴
- **위트 코드** — 유머 한 스푼 (구글리아이, 말장난 비주얼)
- **⚠️ 1차원 직역 금지 (현업 감도)** — 컨셉을 문자 그대로 그리지 말 것. '장벽=벽돌벽·요새·돔' 같은 직역은 유치하다. 대신 ①카피의 말맛·펀을 살리고 ②USP를 **세련된 그래픽 은유**(빛의 기준선·궤도·수평선·등고선)로 은은히 암시한다. 다크 임팩트만이 답 아님 — **밝고 청량한 톤 + 자연스런 인물 + 여백**도 강력. (리얼베리어 런: 벽돌벽 직역=반려, 밝은 궤도필드 은유=채택)
- **레퍼런스는 감도 학습용일 뿐** — 사용자가 레퍼런스를 줬어도 레이아웃·기법(찢기, 손글씨 등)을 그대로 복붙하지 않는다. 오리지널 컨셉으로 재구성.
- 10개 중 결이 다른 3~4개를 사람에게 보여주고 방향 게이트. ("알아서 쭉" 지시가 있었으면 게이트 생략하고 가장 강한 1~2개로 바로 진행.)

### 3. 무드 → 타이포 매핑 (반드시 프롬프트에 명시)
"bold headline/clean sans-serif"만 쓰면 밋밋한 고딕=PPT 텍스트로 나온다. 무드별 서체 처리를 프롬프트에 **아트디렉팅 문구로 직접 지정**한다:

| 무드 | 서체 처리 | 프롬프트 키워드 예 |
|---|---|---|
| 다크·프리미엄 | 크롬/리퀴드 메탈 + 글로우 | "chrome liquid metal typography, specular highlight, dark studio glow" |
| 위트·캐주얼 | 마커 손글씨 | "thick marker handwriting, slightly imperfect strokes, playful" |
| 감성·휴먼 | 붓 캘리그래피 | "brush calligraphy ink stroke, soft bleed edges" |
| 조형·임팩트 | 3D 압출 + 질감 | "3D extruded typography, beveled edges, dramatic rim light, material texture" |
| 미니멀·클린 | 라인과 하나된 얇은 타입 | "thin minimal line type integrated into composition, negative space" |
| 신뢰·정직(금융 등) | 3D 압출 단색(브랜드컬러) | "solid brand-color 3D extruded type, clean matte finish, no gradient" |

이펙트(번개·엠보싱·왜곡·그림자)까지 구체적으로 지정한다.

> **카피 = "얹은 자막"이 아니라 "디자인된 포스터 헤드라인".** 영화 포스터 타이틀처럼 조판돼야 한다. `references/kv-references.md`의 사용자 앵커를 직접 참조:
> - **앵커 1(스트레치·왜곡 세리프)** — 한글 획을 좌우로 늘리고 왜곡해 인물 앞을 가로지름.
> - **앵커 3(찢기)** — 대형 한글이 배경을 번개처럼 찢고 그 틈으로 인물 노출.
> - **앵커 5(벌룬/물질감)** — 글자마다 부푼 3D 질감.
> 밋밋하게 가운데 정렬한 고딕 자막 = 아마추어. 자간·크기 대비·인물과의 겹침·레이아웃을 포스터 디자이너처럼 설계한다.

### 4. 실사 미장센 레버 — AI티 제거가 최우선 (인물 필수)
> **목표: 영화 포스터 / 패션 화보 한 컷.** "AI가 만든 티"가 나면 그 즉시 실패. 특히 인물.
- **AI 렌더 티 금지어를 프롬프트에 명시**: `photographed on film (Kodak Portra / Cinestill), real skin texture with pores and fine lines, natural asymmetry, catchlight in the eyes, no plastic/waxy skin, no over-smoothing, no CGI look, no uncanny valley, not an illustration`
- **영화·화보 기준 문구**: `cinematic movie-poster still` 또는 `high-fashion editorial (Vogue/Dazed) campaign`, 정확한 렌즈·조명·그레이딩 지정 — 85mm f/1.4, soft key + rim light, film grain, 시네마틱 컬러 그레이딩.
- **손·눈·치아·귀** 등 AI가 잘 뭉개는 부위는 "natural realistic hands, correct fingers" 식으로 못 박는다.
- 스톡틱·플라스틱 피부·죽은 눈·완벽 대칭 = 반려. **불완전함(자연광, 실제 모공, 미세한 비대칭)이 곧 리얼리티.**
- 비율 왜곡 금지 — 0단계에서 정한 규격 비율 유지, 셀에 안 맞으면 크롭이지 늘리기 아님.

### 5. 생성 (이미지 생성 커넥터 필요)
> ⚠️ **전제**: Claude 자체는 이미지를 만들지 못한다. 이 단계는 **이미지 생성 MCP/커넥터가 연결된 환경**(nano-banana·Higgsfield·Midjourney 브리지 등)을 전제로 한다.
> - 커넥터가 있으면 → `generate_image` 계열로 **최소 6개 시안** 생성.
> - 커넥터가 없으면 → **완성형 프롬프트 패키지(6벌)까지만 출력**하고, 사용자가 본인 이미지 툴에 붙여넣게 안내한다. (스킬은 여기까지가 자기 역할)
- 2개는 부족 — 반려 사유였음. 무조건 6개 이상.
- **이미지마다 한 줄 정리 필수**: ①어떤 컨셉인지 ②어떤 느낌으로 만들었는지 ③뭘 참고했는지 (레퍼런스가 있었다면).
- 로고 고정 위치(하단 또는 지정 위치) 반드시 합성. KV 하나에 컨셉 하나만 — 컨셉 없는 예쁜 무드샷은 실패.

### 6. 리뷰 게이트
- **⚠️ 결과는 한 장으로 합쳐 보여준다** — 생성한 시안을 개별로 흩뿌리지 말고 **라벨 붙인 컨택트시트(그리드 1장)로 합쳐 로컬 저장 후 `open`** 한다. (채팅 위젯이 안 뜨는 CLI 환경 대비 — 실패 사유였음. PIL로 2×N 그리드 + 시안명 라벨.)
- 사람에게 6개 중 픽 요청 (narrow 게이트 — 컨셉 방향까지 재질문하지 않음, 이미 골랐으면 바로 다음 단계).
- 원본 벤치마크가 있는 런(비딩 제안서 등)이면 **벤치마크의 구조·밀도를 그대로 계승**했는지 자체 점검 — 여백 많은 문서형으로 임의 재구성했으면 반려감.

## 이 스킬이 하지 않는 것
- 카피 완성도 자체(라임·말맛 다듬기)를 깊게 파는 일 → 전용 카피 스킬이 있으면 그쪽에 위임
- 스토리보드 컷 구성·영상 시퀀스 → 별도 진행(단, 같은 무드→타이포 매핑·실사 레버 원칙은 그대로 적용)

## 체크리스트 (출력 전)
- [ ] **0단계 오프닝 게이트**로 사이즈·컨셉·레퍼런스·실사무드/카피수준을 먼저 물었는가
- [ ] **AI티 제거** — 인물이 영화 스틸/화보처럼 포토리얼인가 (플라스틱 피부·죽은 눈·이상한 손 ❌)
- [ ] **카피가 포스터 헤드라인처럼 디자인**됐는가 (가운데 정렬 고딕 자막 ❌)
- [ ] 카피는 확정된 것에서 왔는가, 비주얼 작업 중 즉흥으로 바꾸지 않았는가
- [ ] 컨셉 10개 발산했는가, 레퍼런스 그대로 베끼지 않았는가
- [ ] 최소 6개 시안 + 각 이미지 한 줄 정리
- [ ] 로고 고정 위치 반영했는가
- [ ] 0단계에서 정한 비율 유지(왜곡 ❌)했는가
